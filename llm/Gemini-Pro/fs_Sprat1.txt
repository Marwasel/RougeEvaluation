Summary of "Security and Privacy Requirements Analysis Tool (SPRAT) Software Requirements Specification"
This document describes the functional and non-functional requirements for SPRAT, a tool designed to assist analysts in the scenario and goal mining, reconciliation, and management processes. SPRAT will provide a repository for goals and scenarios derived from policies and other documents, allowing for analysis of privacy and security policies.

Key features of SPRAT include:

User Access Module (UAM): Provides access levels and permissions for different user classes (administrator, project manager, analyst, guest).
Goal Specification and Management Module (GSM): Supports goal management, including adding, deleting, updating, classifying, and viewing goals.
Policy Management Module (PM): Supports policy management within the tool.
Flesch Readability Index Module (FRE): Calculates the Flesch Readability Index of policy documents.
Scenario Specification and Management Module (SSM): Supports scenario management within the tool.
Requirements Specification Module (RS): Supports requirements for a system.
Legal Compliance Module (LC): Ensures compliance with relevant legal regulations.
Requirements-level Access Control Analysis Framework Module (RACAF): Supports RACAF functionality.
The document outlines the specific requirements for each module, including the purpose, description, origin, and priority of each requirement. Additionally, it specifies non-functional requirements related to data security and access control.

The document also includes information about the document revision history, glossary, and references.

Key points:

SPRAT is a tool for managing goals, scenarios, and policies.
It provides different access levels for different users.
It includes modules for managing goals, policies, scenarios, and requirements.
It calculates the Flesch Readability Index of policy documents.
It ensures compliance with legal regulations.
It supports RACAF functionality.
