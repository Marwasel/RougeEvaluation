Summary of Functional Requirements for SPRAT
This document outlines the functional requirements for the Security and Privacy Requirements Analysis Tool (SPRAT), a system designed to assist analysts in the scenario and goal mining, reconciliation, and management processes.

User Access Module (UAM)
FR-UA 1: The system shall support administrator, project manager, analyst, and guest access levels with specific permissions and restrictions for each.
FR-UA 2: The system shall allow project managers to insert, delete, and manage privacy policies, domains, user groups, and project assignments.
FR-UA 3: The system shall allow analysts to add, delete, and update goals, scenarios, requirements, and access control policies.
FR-UA 4: The system shall allow guests to view information with restrictions set by the project manager.
Goal Specification and Management Module (GSM)
FR-GSM 1: The system shall allow analysts to add new goals with detailed information including ID, description, taxonomy, subject classification, actor, policy source, granularity, observability, occurrences, context, and relevant legislation.
FR-GSM 2: The system shall provide templates for goals, scenarios, P3P statements, EPAL rules, and access control policies.
FR-GSM 3: The system shall allow analysts to classify goals based on policy/scenario, observability, protection/vulnerability, and subject classification.
FR-GSM 4: The system shall allow analysts to choose multiple subject classifications for each goal.
FR-GSM 5: The system shall allow the project manager to add new goal classification types.
FR-GSM 6: The system shall allow analysts to request new goal classification types and the project manager to create them.
FR-GSM 7: The system shall allow analysts to update existing goals.
FR-GSM 8: The system shall allow analysts to delete existing goals.
FR-GSM 9: The system shall automatically propagate associated policy information upon deletion or replacement of a goal.
FR-GSM 10: The system shall maintain traceability links between goals and their source policies.
FR-GSM 11: The system shall display the number of occurrences of a goal in a policy, multiple policies, and within a domain.
FR-GSM 12: The system shall display the number of different goals that occur in a policy.
FR-GSM 13: The system shall display the context of a goal.
FR-GSM 14: The system shall allow analysts to create and update goal keyword definitions and allow guests to view them.
FR-GSM 15: The system shall allow the project manager or the definition creator to lock and unlock keyword definitions.
FR-GSM 16: The system shall allow analysts and guests to search goals according to analyst-chosen attributes.
FR-GSM 17: The system shall allow analysts to view elements of a goal returned by a query.
FR-GSM 18: The system shall provide a template to express goals in BNF grammar.
FR-GSM 19: The system shall support conflict identification and resolution between requirements and privacy policies.
FR-GSM 20: The system shall allow analysts to view scenarios associated with individual goals.
FR-GSM 21: The system shall provide a dedicated section for P3P specifications.
FR-GSM 22: The system shall provide a dedicated section for EPAL specifications.
Analysis Document Management Module (ADM)
FR-ADM 1: The system shall allow project managers to add analysis documents and assign domains to them.
FR-ADM 2: The system shall allow project managers to edit document domains.
FR-ADM 3: The system shall allow project managers to delete document domains.
FR-ADM 4: The system shall allow project managers to create document types.
FR-ADM 5: The system shall allow analysts and guests to view documents in alphabetical order within a specific domain.
FR-ADM 6: The system shall allow project managers to create new domains and assign existing documents to them.
FR-ADM 7: The system shall support automatic multi-user analysis results comparison upon project manager request.
Flesch Readability Module (FRE)
FR-FRE 1: The system shall provide a way to calculate the Flesch Readability Index for textual documents.
